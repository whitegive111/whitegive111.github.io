VNCTF 解题记录

发表于2022-02-12|更新于2022-02-28|CTF
|阅读量:

前言

放假在家没事随便玩玩,正好看到了VNCTF,那就打!不过这次Crypto跟二进制也太难了,呜呜呜菜鸡哭哭。最后侥幸进入前十,因此分享个人的Write up供各位师傅们参考交流,如果有什么不正确的地方,还请各位师傅们指点迷津。

WEB

GameV4.0

直接F12搜flag可以在data.js里面搜索到flag的base64形式

解码即可

gocalc0

打开发现是计算器,点击一下flag在这里可以发现cookie中出现session

刚开始base64解码出现了数字+字母的组合,一直感觉方向不对弄不出来

后面尝试将奇怪的数字字母和符号去掉,竟然正确了

newcalc0

只需要污染到原型就行

参考dicectf2022https://brycec.me/posts/dicectf_2022_writeups#vm-calc

发送payload之后访问/flag

Crypto

ezmath

题目要求找出满足第n个满足$2^t \equiv 1 \ (mod \ 15)$形式的数，根据阶的性质，4为满足$2^r \equiv 1 \ mod \ 15$的最小数，说明2模15的阶为4，因此，所有形为4k的数均满足以上条件，因此对于第n个，我们发送4n即可。

Misc

仔细找找

﻿用PS打开，仔细分析下点阵的规律

一开始直接尝试用规律的间隔去取点阵每一个点，后来发现这里并不是规律的，有写地方的间隔会比其他地方的间隔多一个像素点，稍微麻烦了点，所以直接x轴取一列，y轴取一列，把他们的坐标拿出来，然后直接取所有点的转换成图像

Strange flag

追踪http流

可以看到关键信息flag: Is a directory

继续追踪,发现很多文件夹,并且上面有flag字眼

发现在文件目录中有些目录的长度不同,因此尝试将前面十六个7级目录中，还有下级文件夹的视为1，没有的则视为0，转换成二进制，发现前十六位二进制转换字符确实有字符的，因此可以根据每行字符串长度进行转换

发现这里得到的并不是flag,而是;类似于栅栏加密的flag,但似乎也不完全是栅栏,很容易发现v,n,c,t之间都夹着10个字符,把v,n,c提取出来,与后面的tf{连起来,然后依照顺序连接即可。

问卷

全是好题!

prize wheel

竟然有概率得到zip,那就狂点1就好了(事实证明我的运气还比较好,点了40+次吧,99.5%^45=0.7980)

可以看到图片有非常多的正方形框,像是将图片的像素点依照正方形框按照某个方向移动。因为图像是不规则的图像,因此应该是部分图像进行了转换,而部分并没有转换。经过多次尝试发现正方形框的移动方向为逆时针方向

现在的重点就是怎么进行定位哪些正方形框需要进行变换而哪些不需要,我们可以观察到在图片的中心区域有很多白点,而且题目提示也说了图片中有定位的地方,因此尝试了多次,最终发现y轴最中心处可以作为定位依据,根据每个像素点的值是否为(255,255,255,255)

simple macos

给了个mac系统的文件,可惜没有什么好的取证工具,只能手点随便乱看文件。

最后在simple macos\System\Volumes\Preboot\79FABCCE-3636-4266-A6CF-8E3BB40332B4\var\db下发现一个base串,很明显是jpg图片的base64形式

看到文件尾有段附加数据跟类似于flag的尾部。根据前面的提示可以知道是oursecret隐写,但是要注意那部分类似于flag的尾部内容应该不是oursecret附加的,因此解密时需要删除

然后根据提示也可以知道是弱口令,直接乱填(123,1234,12345)结果就是123456。
